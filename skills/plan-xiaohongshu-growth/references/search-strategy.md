# Xiaohongshu Search Strategy

## Intent map

Classify the reader's immediate job:

| Intent | Typical language | Useful content |
| --- | --- | --- |
| Learn | 是什么、为什么、教程 | Explanation plus steps |
| Compare | 对比、区别、怎么选 | Criteria and trade-offs |
| Solve | 没流量、限流、转化低 | Diagnosis and checklist |
| Execute | SOP、模板、清单 | Sequence, owner, acceptance rule |
| Validate | 案例、实测、复盘 | Method, evidence, limits |

Choose one primary keyword that names the problem or category in the user's language. Add two to five supporting expressions only when they narrow audience, scenario, outcome, or method.

## Placement

- Title: use the primary term naturally when it does not damage clarity.
- Opening: confirm the reader's scenario and problem.
- Body: use variants only where the section genuinely answers them.
- Ending: summarize the decision or next action; do not repeat keywords mechanically.
- Tags: add a small set of relevant category and scenario tags. Do not use unrelated trending terms.

## Evidence rules

Do not claim search volume, competition, rankings, index inclusion, or algorithm weight without current evidence. Search suggestions and visible result patterns are qualitative signals, not official demand numbers.
