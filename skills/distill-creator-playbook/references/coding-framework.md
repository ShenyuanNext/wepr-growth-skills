# Creator Content Coding Framework

Use a consistent ledger for every sample:

| Dimension | Record |
| --- | --- |
| Evidence | URL/ID, date, accessibility, capture time |
| Audience | Who is addressed and at what awareness stage |
| Job | Problem solved or progress enabled |
| Topic | Pillar, subtopic, scenario, freshness |
| Packaging | Title promise, cover device, opening hook |
| Structure | Story, checklist, tutorial, comparison, case, opinion |
| Proof | Demonstration, data, artifact, experience, citation, none |
| Emotion | Curiosity, relief, fear, identity, ambition, disagreement |
| CTA | Save, comment, follow, profile, conversation, purchase |
| Signals | Views, likes, saves, comments, questions, objections |

## Evidence strength

- **High:** directly observable repeated pattern across comparable samples.
- **Medium:** observable pattern with limited sample or meaningful confounders.
- **Low:** plausible inference requiring testing.

Never convert correlation into a causal claim. Record alternative explanations.
